# 🚀 نظام الـ Server Actions مع Form Validation (React 19)

هذا الدليل يشرح كيفية دمج **Next.js Server Actions** مع مكتبة **React Hook Form** و **Zod** لضمان أفضل تجربة مستخدم (UX) وأعلى مستويات الأمان (Security).

## 🏗️ المكونات الأساسية (The Architecture)

1.  **Server Action (`_actions/index.ts`)**: الفانكشن اللي بتشتغل على السيرفر وبتعمل معالجة للبيانات (مثل الحفظ في قاعدة البيانات).
2.  **Generic Hook (`_hooks/index.ts`)**: هوك `useServerForm` اللي بيربط بين حالة السيرفر وحالة الفورم في المتصفح.
3.  **UI Component (`ServerForm.tsx`)**: الواجهة اللي بيشوفها المستخدم.

---

## ❓ أسئلة وأجوبة للتوضيح (Q&A for the Team)

### س: ليه بنعمل فاليشن مرتين (مره في الـ Client ومره في الـ Server)؟

**ج:**

- **في الـ Client:** عشان السرعة (UX). المستخدم يعرف غلطه فوراً بدون انتظار رد السيرفر.
- **في الـ Server:** عشان الأمان (Security). أي حد يقدر يبعت طلبات للسيرفر مباشرة بدون ما يمر بالمتصفح، فالسيرفر لازم يتأكد بنفسه من صحة البيانات.

### س: إزاي الـ Server Action بيعرف يرجع الأخطاء لكل حقل لوحده؟

**ج:** الـ Action بيرجع Object فيه خاصية `errors` على شكل `{ fieldName: ['error message'] }`. الهوك بتاعنا (`useServerForm`) بياخد الـ Object ده ويعمل له Mapping داخل الفورم باستخدام `form.setError`.

### س: ليه الـ `onSubmit` في المكون فاضية `onSubmit={() => {}}`؟

**ج:** لأننا في رياكت 19 بنعتمد على الـ `action` البروب الأساسي للفورم. الـ `onSubmit` هنا وظيفتها بس إنها تشغل الـ `handleSubmit` وتمنع الإرسال لو فيه أخطاء في الـ Client. بمجرد ما الفلديشن ينجح، رياكت بتكمل أوتوماتيكياً وتنفذ الـ `action`.

## 🛠️ كيفية الاستخدام في أي صفحة جديدة

1.  عرف الـ **Zod Schema**.
2.  اعمل الـ **Server Action** واستخدم `formSchema.safeParse`.
3.  نادي هوك `useServerForm` وابعتله السكيما والأكشن.
4.  استخدم الـ `action` و الـ `form` اللي راجعين من الهوك في الـ `SchemaForm`.
